Fonctionnalités
Ce que fait l'afficheur local vs ac.hiboutik.net
🎟
Fin de vente automatique
QR code reçu numérique + QR code concours pré-rempli avec le numéro de ticket, affichés automatiquement à chaque clôture de vente.
⏱
Durée paramétrable
Le ticket reste affiché le temps configuré (15s–120s). Retour automatique au mode idle jusqu'à la prochaine vente.
🎞
Slideshow ou vidéo
Entre les ventes : carousel d'images ou vidéo en boucle. Glisser-déposer depuis le PC local — aucun upload cloud nécessaire.
⚙️
Interface admin locale
Tout se configure depuis localhost:3000/admin sur le PC. Drag & drop des médias, toggles, sliders. La tablette ne voit que /display.
🔒
Données 100% locales
Le serveur interroge directement l'API Hiboutik du franchisé. Aucune donnée ne transite par les serveurs LMDV.
📱
Anti-veille tablette
Screen Wake Lock API (browser) ou wrapper Capacitor Android pour une tablette qui ne s'éteint jamais.

# LMDV Afficheur Local — Session de développement Claude Code ## Contexte projet Tu développes "LMDV Afficheur Local", une application d'afficheur client de caisse pour les boutiques franchisées La Maison du Vapoteur. Ce projet remplace l'afficheur natif Hiboutik (ac.hiboutik.net) par une solution 100% locale et personnalisable. ## Décisions d'architecture (non négociables) - Polling (pas de webhook) : le serveur interroge GET /sales/ toutes les 3 secondes - Détection par comparaison de sale_id (lastSaleId vs nouvelle réponse) - WebSocket local ws://localhost:PORT pour pousser les événements vers la tablette - Admin sur /admin accessible uniquement depuis localhost (le PC du gérant) - Tablette accède uniquement à /display — jamais à /admin - config.json surveillé par chokidar → rechargement à chaud sans restart - Zéro framework front (Vanilla JS uniquement côté client) - Zéro dépendance aux serveurs LMDV ## Stack - Node.js 18+ + Express + ws + multer + chokidar + node-fetch - HTML5 / CSS3 / Vanilla JS côté client (display ET admin) - Screen Wake Lock API dans display.js ## Fichiers à créer ### server.js Serveur Express complet avec : - Polling Hiboutik : GET https://{account}.hiboutik.com/api/sales/?store_id={store_id} Auth : Authorization: Basic base64(email:api_key) Normalisation : Array.isArray(data) ? data[0] : data Trier par completed_at DESC, garder le plus récent, comparer avec lastSaleId Si nouveau → broadcast JSON {type:'sale_closed', sale} à tous les WS clients - WebSocket server sur même port (/ws) - Serve public/ en statique - Admin API endpoints : GET /api/config → retourner config sans hiboutik.api_key POST /api/config → sauvegarder config.json (chokidar recharge auto) POST /api/media/upload → Multer single file → sauvegarder dans public/media/ DELETE /api/media/:filename → supprimer public/media/{filename} GET /api/media → lister public/media/ (jpg, png, webp, mp4, webm) POST /api/preview → broadcast WS {type:'sale_closed', sale: FAKE_SALE} GET /api/status → {polling: true, lastSaleId, connectedClients, uptime} - Chokidar watch config.json → recharger la config en mémoire sans restart - Gestion d'erreur robuste : si API Hiboutik indisponible, logger et continuer le polling ### public/display.html + css/display.css + js/display.js Page afficheur tablette avec 3 états : ÉTAT IDLE : - Plein écran, fond sombre (#05090f ou config couleur) - Carousel d'images (config.media.items) avec transition CSS fade/slide OU vidéo HTML5 autoplay loop muted (config.media.video_path) - Texte d'accueil centré (config.display.idle_message) - Horloge HH:MM:SS en haut à droite ÉTAT VENTE (déclenché par WS sale_closed) : - Animation d'entrée (slide up ou fade) - Layout 2 colonnes : Gauche : QR code → URL myrecei.pt du ticket Généré via https://api.qrserver.com/v1/create-qr-code/?size=220x220&data=URL_ENCODEE Si sale.qr_code disponible, utiliser cette URL, sinon construire depuis sale_id Numéro ticket visible : "#{sale.sale_id}" Si sale.unique_sale_id : afficher aussi le format 2026-06-2-151 Droite (si config.display.show_concours_block) : QR code → {config.display.concours_url}?ticket={sale.sale_id} Label "Participez au concours !" Couleurs LMDV : #FF766C (corail), #003D94 (cobalt), #1C2135 (fond) - Footer : total TTC + mode de paiement - Timer : après config.display.sale_duration_seconds → transition retour IDLE RECONNEXION WS : - Si WS déconnecté, retry toutes les 3 secondes - Indicateur de connexion discret (point vert/rouge en bas de page) WAKE LOCK : - navigator.wakeLock.request('screen') au chargement - Re-request sur document visibilitychange (revient au premier plan) - Catch silencieux si non supporté Paramètre URL ?preview=true : simuler une vente fictive au chargement (pour test sans serveur) ### public/admin.html + css/admin.css + js/admin.js Interface admin sur /admin avec sidebar + 4 onglets : SIDEBAR : - Logo LMDV, liens vers les 4 onglets - Indicateur statut serveur (polling actif, nb clients WS connectés) ONGLET 1 — Médias idle : - Sélecteur type : "Slideshow" ou "Vidéo" (boutons toggle) - Mode Slideshow : Zone drag & drop (dragover/drop events) qui envoie POST /api/media/upload Barre de progression pendant l'upload Grille des médias uploadés avec numéro d'ordre, bouton supprimer (DELETE /api/media/:file) Drag pour réordonner la liste (drag and drop natif sur les items) Slider "Durée par slide" (2s–15s) - Mode Vidéo : Zone drag & drop pour uploader un .mp4 ou .webm Aperçu du fichier sélectionné - Bouton "Enregistrer" → POST /api/config - Bouton "Aperçu sur tablette" → POST /api/preview ONGLET 2 — Affichage vente : - Toggle ON/OFF "Bloc concours" (show_concours_block) - Input URL concours (modifiable sans redémarrage) - Slider "Durée d'affichage" (15s–120s, step 5s) avec valeur en direct - Bouton "Simuler une vente" → POST /api/preview - Bouton "Enregistrer" ONGLET 3 — Boutique : - Input account Hiboutik - Input email - Input clé API (type=password, bouton show/hide) - Input store_id - Input port serveur - Bouton "Tester la connexion" → vérifie GET /sales/ et affiche OK/Erreur - Bouton "Enregistrer" ONGLET 4 — Aperçu : - Iframe pointant sur /display?preview=true - Bouton "Simuler une vente" sur la page admin qui envoie POST /api/preview → le display dans l'iframe reçoit le WS et affiche l'état vente Design admin : - Fond #05090f, card #0b1420, couleurs LMDV - Font Montserrat (Google Fonts) - Sobre et professionnel, pas de fioritures ### config.example.json { "hiboutik": { "account": "moncomptehiboutik", "email": "gerant@franchise.fr", "api_key": "VOTRE_CLE_API", "store_id": 1 }, "display": { "sale_duration_seconds": 45, "poll_interval_ms": 3000, "idle_message": "Bienvenue à La Maison du Vapoteur", "concours_url": "https://lmdv-concours.vercel.app", "show_concours_block": true }, "media": { "type": "slideshow", "items": [], "slide_duration_seconds": 6, "video_path": "" }, "server": { "port": 3000 } } ### package.json { "name": "lmdv-afficheur-local", "version": "1.0.0", "type": "module", "scripts": { "start": "node server.js", "dev": "nodemon server.js" }, "dependencies": { "chokidar": "^3.5.3", "express": "^4.18.2", "multer": "^1.4.5-lts.1", "node-fetch": "^3.3.2", "ws": "^8.16.0" }, "devDependencies": { "nodemon": "^3.0.2" } } ### README.md (court) Installation en 5 étapes, accès tablette, lien vers /admin ## Qualité attendue -
